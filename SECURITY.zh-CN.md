# 安全策略

[English](./SECURITY.md) | [简体中文](./SECURITY.zh-CN.md)

## 支持版本

AgentX 目前仍处于早期开发阶段。

当前只有最新开发线会被视为安全修复支持版本：

| 版本 | 是否支持 |
| --- | --- |
| `0.1.x` | 是 |
| `< 0.1.0` | 否 |

## 漏洞报告

请不要通过公开 GitHub issue 报告安全漏洞。

请改用以下私密渠道之一：

- GitHub 私有漏洞报告功能（如果仓库已启用）
- 通过私密渠道直接联系仓库维护者

提交漏洞报告时，建议尽量包含：

- 受影响的组件或功能
- 复现步骤或 PoC
- 影响评估
- 如有可能，附带修复建议

我们会尽量：

- 尽快确认已收到报告
- 验证问题是否存在
- 评估严重程度和影响范围
- 在合适的时候准备并发布修复
- 在给用户留出升级时间后再负责任地披露修复信息

## 关注范围

以下类型的问题尤其值得报告：

- 认证与授权
- JWT 处理和会话行为
- 资源包上传与解压
- 本地安装路径处理
- 任意命令执行
- 持久化导入导出流程
- 依赖与供应链风险

## 开发说明

为了便于本地测试，本项目当前内置了演示账号和本地开发默认值。它们仅用于本地开发与演示，绝不能视为生产可用默认配置。

生产部署至少应设置：

```bash
AGENTX_JWT_SECRET="change-me-in-production"
AGENTX_DEMO_PASSWORD="replace-demo-password"
```
